三种工厂模式

发表于2022-06-15|更新于2022-06-15|设计模式
|阅读量:

三种工厂模式

  • 简单工厂模式
  • 工厂方法模式
  • 抽象工厂模式

简单工厂模式

每次要添加一种动物类，工厂就要多加一个 type。导致这个工厂类会变得非常繁重，甚至加入的这个 type 也可能再有变动。

例如:一个动物基类(抽象基类):描述什么是动物
N 个动物类(继承抽象类):具体是什么动物,狗猫兔
一个工厂(根据不同条件生成创建不同动物):

new cat ->猫
new dog ->狗

工厂方法模式

每次生产一种新种类的动物，只需要再添加一个工厂就可以了。
耦合性比起简单模式要松很多。这个工厂使用太单一，一种工厂只能生产一种动物。

例如:一个动物基类(抽象基类):描述什么是动物
N 个动物类(继承抽象类):具体是什么动物,狗猫兔
一个抽象工厂接口(接口)
创建生产动物的工厂实现类(实现接口)

抽象工厂模式

依据“产品族”区分工厂，最大限度的进行抽象，可以让这个工厂实现各种花样的功能，但同时每次增加一种类型的工厂，意味这从工厂到产品基类都要跟随变动。

例如:一个动物基类(抽象基类):描述什么是动物
N 个动物类(继承抽象类):具体是什么动物,狗猫兔
一个染色接口(接口):描述可以染什么色
N 个染色类(继承抽象类):具体染什么颜色:红绿蓝
一个抽象工厂接口(接口):里面定义动物基类和染色基类
创建生产动物并染色的工厂实现类(实现接口):生产什么动物,染什么色
