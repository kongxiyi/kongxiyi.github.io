关于类型转换

发表于2022-05-16|更新于2022-06-09|js基础
|阅读量:

概述

有三种常用的类型转换：

  • 转换为 string 类型
  • 转换为 number 类型
  • 转换为 boolean 类型

字符串转换

字符串转换 —— 转换发生在输出内容的时候，也可以通过 String(value) 进行显式转换，也可以 （a+’’） 进行转换

数字型转换

数字型转换 —— 转换发生在进行算术操作【- 、 / 、 * 、 % 、 **】时，注意无 + ，也可以通过 Number(value) 进行显式转换。

值 变成…
undefined NaN
null 0
true / false 1 / 0
string “按原样读取”字符串，两端的空白会被忽略。空字符串变成 0。转换出错则输出 NaN。

布尔型转换

布尔型转换 —— 转换发生在进行逻辑操作时，也可以通过 Boolean(value) 进行显式转换。

布尔型转换遵循以下规则：

值 变成…
0, null, undefined, NaN, “” false
其他值 true

上述的大多数规则都容易理解和记忆。人们通常会犯错误的值得注意的例子有以下几个：

  • 对 undefined 进行数字型转换时，输出结果为 NaN，而非 0。
  • 对 “0” 和只有空格的字符串（比如：” “）进行布尔型转换时，输出结果为 true。
