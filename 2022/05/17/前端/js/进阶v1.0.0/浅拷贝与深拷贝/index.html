浅拷贝与深拷贝

发表于2022-05-17|更新于2022-06-10|js基础
|阅读量:

概念

  • 值拷贝: 基本数据类型 ( String 、Number 、Boolean 、Null 、Undefined ) 在赋值的过程中都是值拷贝.
  • 浅拷贝：拷贝引用
  • 深拷贝：复制对象的属性给新对象。

js 中所有的原始数据类型默认执行深拷贝，比如 Boolean, null, Undefined, Number,String 等，也就说法是叫做值拷贝

针对于对象的浅拷贝

  • 直接赋值

Object 的深拷贝

  • 自己写遍历
  • Object.assign
  • spread 语法
    • 如：let objCopy = { …obj }
  • 第三方框架，如 lodash 库的 _.cloneDeep(obj)

使用 const 声明的对象也是可以被修改的

看起来 (*) 行的代码会触发一个错误，但实际并没有。user 的值是一个常量，它必须始终引用同一个对象，但该对象的属性可以被自由修改。

换句话说，只有当我们尝试将 user=… 作为一个整体进行赋值时，const user 才会报错。

Array 的深拷贝

  • 自己写遍历
  • slice
  • concat
  • spread 语法
    • 如：let arrCopy = [ …arr ]
  • 第三方框架

万能转换器（对 Array 和 Object 等都适用）

  • JSON.parse(JSON.stringify(obj))

前面讲解了 Array 和 Object 的深拷贝方法，但是对于有更深层次的结构关系（数组套数组 数组套对象 对象套对象等），上面的方法就失灵了。

JSON.parse(JSON.stringify(obj))深拷贝已有对象，它可以深拷贝多层级的，不用担心嵌套问题。
